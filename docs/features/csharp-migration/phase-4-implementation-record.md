## Approved Deviations

### D-1: No start-point parameter in CheckoutAsync
`IVersionControl.CheckoutAsync(create: true)` branches from current HEAD. PowerShell uses `git checkout -b $branch $baseBranch` with an explicit start point. The C# approach requires the caller to position to the base branch first (documented in `<remarks>`).

### D-2: Runtime state preservation deferred to Phase 7+
PowerShell `Apply-Suggestion.ps1` preserves the results directory across branch switches. This is deferred because the results-directory concept is not yet migrated. Tracked via `TODO(phase-7+)` in `ExperimentBranchManager.cs`.

### D-3: Artifact staging deferred to Phase 7+
PowerShell `Stage-ExperimentArtifacts.ps1` stages analysis/iteration/metrics files before commits. This requires the full experiment artifacts model (Phase 7+). Tracked via `TODO(phase-7+)`.

### D-4: Branch push after revert not included
PowerShell `Revert-ExperimentCode.ps1` calls `Invoke-ExperimentBranchPush` after the revert commit. The C# `RevertExperimentAsync` only handles the local revert â€” pushing is an orchestrator-level concern handled separately.

### D-5: SoftReset retry path not implemented
PowerShell `Revert-ExperimentCode.ps1` has a dedicated `$SoftReset` switch for retry scenarios. Only the full-revert path is implemented. The retry path will be needed when the iterative fix loop is migrated.

## Risks

1. **Runtime state preservation gap** â€” Until Phase 7+ migrates the results-directory concept, branch switches during experiments may lose runtime artifacts. The PowerShell path preserves them.

2. **Partial failure cleanup** â€” Both `ApplySuggestionAsync` and `RevertExperimentAsync` can leave the repo in a dirty state on partial failure (e.g., branch created but commit failed). This matches PowerShell behavior but should be addressed with rollback logic in a future hardening pass.

3. **JSON error parsing in GitHubCodeHost** â€” `GetPullRequestStatusAsync` parses `gh` output as JSON without wrapping parse exceptions. A non-JSON error response (e.g., auth failure) produces a raw `JsonException` rather than a contextualized error.
