Migration Overview

From — PowerShell Harness

  • ~11,000 lines across 87 files
  • Dynamic typing throughout
  • Poor testability & limited IDE support
  • Monolithic script architecture

To — C# / .NET 10

  • 15 source projects, strong typing
  • Async-first design throughout
  • Plugin architecture via interfaces
  • Independently testable components

Strategy

Clean break — not a line-by-line port. Each phase is independently buildable and testable, with clear interfaces defining the contracts between components. The migration proceeds bottom-up, from core models to full orchestration.

The Agentic Loop

Build→ Verify→ Measure→ Analyze→ Classify→ Implement→ Compare→ Publish→ Repeat
