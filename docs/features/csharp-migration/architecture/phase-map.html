Per-Phase Deliverables

Phase 0 Solution Scaffolding

Projects created: All 15 src + 15 test + 1 shared test infra (31 total)

Key deliverables:

  • Hone.slnx — .NET 10 XML solution format
  • Directory.Build.props — TreatWarningsAsErrors, AnalysisLevel=latest-all, nullable enabled, ImplicitUsings
  • Directory.Packages.props — Central NuGet version management
  • .editorconfig — Naming conventions, formatting rules, analyzer severities
  • BannedSymbols.txt — DateTime→DateTimeOffset, sync File I/O banned
  • Placeholder.cs in each non-Core project

Test infrastructure: HoneTestBase (TempDir, fixture copy, git repo init), GitTestRepo, TargetBuilder

Validation: All builds zero-warning, 15 placeholder tests pass

Files: ~72 files, ~2,000 lines

Phase 1 Core Models, Config & Observability

Projects implemented: Hone.Core (77 C# files)

Key deliverables:

  • 25 domain model records + 4 enums (Models/)
  • 16 config records + ConfigLoader + ConfigMerger (Config/)
  • 9 interfaces + 12 DTOs (Contracts/)
  • HoneEvent hierarchy (7 event types) + HoneEventBus + 2 sinks (Observability/)
  • JsonUtils + StringUtils (Utilities/)

Design decisions: Contracts in Core to prevent circular deps; YamlDotNet with custom RecordAwareObjectFactory; thread-safe HoneEventBus with Lock

Tests: 177 passing (93 model, 20 contract, 18 utility, 25 config, 21 observability)

Files: ~68 files, ~3,500 lines

Phase 2 Measurement & Comparison

Projects implemented: Hone.Measurement (10 files), Hone.Measurement.K6 (5 files), Hone.Measurement.DotnetCounters (7 files)

Key deliverables:

  • MetricComparer — pure function comparing MetricSets with tolerance logic
  • K6LoadTestRunner — wraps k6 CLI, parses JSON summary → MetricSet
  • K6SummaryParser — handles NaN/Infinity edge cases
  • DotnetCountersRuntimeMetricsCollector — wraps dotnet-counters CLI
  • ScaleTestOrchestrator — warmup + N measured runs + median selection
  • BaselineMeasurer — initial performance baseline capture

Tests: snapshot validation for metric comparison, k6 parsing edge cases

Phase 3 Lifecycle & Hooks

Projects implemented: Hone.Lifecycle (18 files)

Key deliverables:

  • HookResolver — maps stage names ("build", "start", "stop", "test") to ILifecycleHook implementations
  • LifecycleHookDispatcher — executes hooks in order with timeout/cancellation
  • DotnetBuildHook — dotnet build via IProcessRunner
  • DotnetStartHook — dotnet run with process lifetime management, returns BaseUrl
  • DotnetStopHook — graceful process shutdown
  • DotnetTestHook — dotnet test with pass/fail parsing
  • HealthPollHook — HTTP polling until health endpoint returns 200
  • K6RunHook — k6 load test as a lifecycle hook
Phase 4 Source Control

Projects implemented: Hone.SourceControl (16 files), Hone.SourceControl.Git (5 files)

Key deliverables:

  • GitVersionControl — IVersionControl implementation wrapping git CLI
  • GitHubCodeHost — ICodeHost implementation using gh CLI
  • ExperimentBranchManager — branch creation, naming (hone/experiment-N), stacked diffs
  • PullRequestManager — PR creation with body generation, labels, draft support
Phase 5 AI Agent Integration

Projects implemented: Hone.Agents.Core (7 files), Hone.Agents.Loop (12 files), Hone.Agents.Preparation (8 files), Hone.Agents.CopilotCli (4 files)

Key deliverables:

  • CopilotCliAgentRunner — IAgentRunner impl invoking copilot CLI
  • ModelDefaults — default model selection per agent role
  • AgentAnalysisContext — shared context for agent invocations
  • AnalysisAgent — generates Opportunity[] from perf data
  • ClassificationAgent — scope classification (narrow vs architecture)
  • ImplementerAgent — code change generation
  • CompatibilityAgent — pre-probe compatibility checks
  • PreProber — pre-experiment health checks
Phase 6 Diagnostic Profiling

Projects implemented: Hone.Diagnostics (29 files)

Key deliverables:

  • PluginDiscoveryService — discovers collector/analyzer plugins from YAML metadata files
  • DiagnosticCollectionOrchestrator — coordinates collectors and analyzers
  • DiagnosticMeasurementOrchestrator — diagnostic-mode measurement (collect during load test)
  • PerfViewCpuCollector, PerfViewGcCollector — ICollectorPlugin impls using PerfView.exe
  • DotnetCountersCollectorPlugin — ICollectorPlugin impl
  • CpuHotspotsAnalyzer — IAnalyzerPlugin analyzing CPU traces
  • MemoryGcAnalyzer — IAnalyzerPlugin analyzing GC traces
Phase 7 Reporting & Export

Projects implemented: Hone.Reporting (19 files)

Key deliverables:

  • HtmlDashboardGenerator — generates HTML dashboard from RunMetadata
  • ResultsRenderer — console table formatting for experiment results
  • ReportFormatter — markdown report generation (RCA, PR body)
Phase 8 Orchestration

Projects implemented: Hone.Orchestration (21 files) — the largest phase

Key deliverables:

  • HoneLoopRunner — main experiment loop controller
  • OptimizationQueueManager — queue lifecycle (create, dequeue, update status)
  • IterativeImplementerRunner — retry loop (implement → build → test → accept/retry/reject)
  • ExperimentRunContext — per-experiment state container
  • ExperimentFailureHandler — failure classification and recovery
  • ArtifactStager — experiment artifact management
Phase 9 CLI Host & Integration Tests

Projects implemented: Hone.Cli (9 files)

Key deliverables:

  • System.CommandLine setup with commands: run, validate, baseline, results, dashboard
  • DI container composition (wires all interfaces to implementations)
  • Common flags: --target, --max-experiments, --dry-run, --model
  • Integration test fixtures
Phase 10 Target Migration & Cutover

Key deliverables:

  • Convert sample-api .psd1 → .hone/config.yaml
  • Documentation updates
  • PowerShell harness archival
  • Final validation: C# harness produces identical results to PowerShell
