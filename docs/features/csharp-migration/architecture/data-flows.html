1. The Agentic Loop — Sequence Diagram

The core experiment loop drives Hone's autonomous optimisation cycle. Each numbered phase feeds its output model into the next.

2. Config Loading Pipeline

Configuration is resolved through a three-layer merge: engine defaults → target overrides → CLI flags. The final HoneConfig is immutable for the duration of the run.

3. Observability Event Flow

All subsystems emit strongly-typed events onto a shared HoneEventBus. Registered sinks consume these events for console output, structured logging, or future integrations.

4. Collector Plugin Lifecycle

Runtime metrics collectors follow a strict Start → Collect → Stop → Export → Analyze lifecycle, managed by PluginDiscoveryService.

5. Data Model Flow

A summary of which domain models are produced and consumed at each phase. Data flows strictly downward — each step feeds the next.
