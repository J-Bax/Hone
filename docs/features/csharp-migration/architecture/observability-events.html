Observability & Event System

Structured event hierarchy, pipeline architecture, sink contracts, and thread-safety model for the Hone C# migration.

1. Event Type Hierarchy

All observability flows through a single abstract base class. Each concrete subtype captures one semantic concern and is color-coded by category.

2. Event Pipeline

Events flow from emitting sources through a centralized bus to all registered sinks. The bus is the single broadcast point; sinks are decoupled consumers.

3. Per-Event-Type Reference

Each card describes when the event fires, its fields, console appearance, and structured JSON shape.

5. Thread-Safety Model

Lock Strategy

HoneEventBus uses System.Threading.Lock — the modern .NET lock type — to protect the internal sink list. The pattern is copy-on-iterate:

  1. Acquire lock → copy the sink list to a local array
  2. Release lock
  3. Iterate the local copy, calling Emit on each sink outside the lock

This eliminates deadlock risk: no user code runs while the lock is held.

Fail-Silent Per Sink

Each sink's Emit call is wrapped in a try/catch. If one sink throws, the exception is swallowed and the remaining sinks still receive the event. One bad sink never breaks the pipeline.

Synchronous Emission

The void Emit(HoneEvent) signature is intentionally synchronous. This keeps ordering guarantees simple — events arrive in the order they were emitted, with no interleaving from concurrent async completions.

⚠ Future consideration: If async sinks are needed (e.g., HTTP webhook, database writes), the recommended pattern is a System.Threading.Channels.Channel<HoneEvent> buffer per sink, with a dedicated consumer task draining the channel. This preserves the synchronous Emit contract while allowing sinks to process asynchronously.
Pseudocode: HoneEventBus.Emit 
public void Emit(HoneEvent honeEvent)
{
    IHoneEventSink[] snapshot;
    lock (_lock)
    {
        snapshot = _sinks.ToArray();
    }

    foreach (var sink in snapshot)
    {
        try
        {
            sink.Emit(honeEvent);
        }
        catch
        {
            // Swallowed — fail-silent per sink
        }
    }
}

6. Extensibility — Adding a New Sink

Three Steps

  1. Implement IHoneEventSink — single method: void Emit(HoneEvent honeEvent)
  2. Register with HoneEventBus.Register(sink)
  3. Pattern-match on HoneEvent subtypes in your implementation

Implementation Template

public class MyCustomSink : IHoneEventSink
{
    public void Emit(HoneEvent honeEvent)
    {
        switch (honeEvent)
        {
            case PhaseStarted ps:
                // handle phase start
                break;
            case PhaseCompleted pc:
                // handle phase completion
                break;
            case MetricSnapshot ms:
                // handle metrics
                break;
            case AgentInvoked ai:
                // handle agent invocation
                break;
            case ExperimentOutcomeEvent eo:
                // handle experiment outcome
                break;
            case StatusMessage sm:
                // handle status message
                break;
            case DiagnosticProgress dp:
                // handle diagnostic progress
                break;
        }
    }
}
